Additional Vocabulary: Unit 1 - Data

  • bit: A binary digit, either 0 or 1.
  • byte: A sequence of 8 bits.
  • roundoff: Error resulting when bits aren't enough to represent a number precisely (e.g., using 3 digits to represent 3.14).
  • analog data: Values that change smoothly over time, like pitch and volume of a live concert.
  • lossless: Compressing data without losing any information, allowing full recovery.
  • lossy: Compressing data by discarding some information, making full recovery impossible.
  • metadata: Data about data, such as descriptive info about a file or database row.

Additional Vocabulary: Unit 2 - Computer Systems and Networks

  • Computing Device: Physical device that runs programs (computer, smartphone, sensor).
  • Computer Network: Group of connected devices sending/receiving data.
  • Path: Series of connections between devices from sender to receiver.
  • Protocol: Rules that govern how systems communicate.
  • Fault Tolerant: Ability to keep working despite component failures.
  • Scalability: Ability to expand or contract system capacity.
  • Bandwidth: Max data transfer rate, measured in bits per second.
  • URL (Uniform Resource Locator): Easy-to-remember web address (e.g., www.example.com).

Internet Structure and Layers

  • Physical Network: Hardware and connections carrying internet signals.
  • Internet Protocol (IP): Assigns unique addresses and routes data.
  • Transmission Control Protocol (TCP): Ensures reliable, ordered, error-checked data delivery.
  • User Datagram Protocol (UDP): Faster, lightweight alternative with minimal error-checking.
  • Packets: Small data chunks for transmission, may arrive out of order or be lost.
  • Routers: Computers forwarding data choosing fastest paths.
  • Domain Name System (DNS): Translates domain names into IP addresses.
  • Hypertext Transfer Protocol (HTTP): Protocol for web pages and media transfer.
  • World Wide Web (WWW): Linked pages and media browsable over HTTP.

Streaming Technology

Streaming technology allows users to watch videos, listen to music, or view other media content in real time without having to download entire files before starting. It works by transmitting data in a continuous flow. It breaks the video into small packets which are sent over the internet and played as they arrive.

Services like Netflix and YouTube use adaptive bitrate streaming, where the video quality adjusts based on the viewer’s internet speed. This relies on network servers called CDNs (Content Delivery Networks) distributed globally to provide videos efficiently with minimal buffering and more.

Key Concepts

  • Message Delivery on the Internet: Messages split into packets taking different paths; no direct connection needed.
  • Redundancy: Multiple network paths so packets can reroute if one fails.
  • Fault Tolerance: Internet keeps working despite damage, thanks to redundancy.
  • Scalability: Designed to support billions of devices without redesign.
  • IP Addressing and Encoding: IPv4 uses 32-bit addresses (~4.3 billion unique); IPv6 expands address space 65,536 times more.
